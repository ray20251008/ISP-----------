# 班會紀錄網頁系統：版本備份與還原規則說明

為了確保您的系統在未來進行任何修改時，都能**安全回退至舊版本**且不被新版本覆蓋，請遵循以下規則與修改流程：

---

## 📋 規則一：手動代碼備份流程（免 Git，最直觀）

當您未來需要對網頁的版面、功能或樣式進行修改時，請依循以下步驟：

### 1. 修改前的備份步驟
在進行任何新修改前，請先將 `C:\Users\admin\Desktop\班會紀錄\class_meeting_record.html` 複製一份，並存入桌面上的 **`歷史版本`** 資料夾中，重新命名為：
`v[最新版本號]_[簡短修改描述]_[日期].html`
> **目前已為您存檔的歷史版本：**
> * 📄 `v1_原始雙欄互動版.html`：包含雙欄介面、基本資料與 Word 匯出。
> * 📄 `v2_防切割與文字修改版.html`：修復了 PDF 列印切頁問題、新增了版面文字直接點擊編輯功能。
> * 📄 `v3_四人自決與手寫分工版.html`（目前運作中）：新增 4 位學生個人決定卡片、點心分工改為手寫/打字輸入、新增一鍵開啟對話歷史按鈕。

### 2. 功能滿意度確認
* 💻 **滿意新版本**：繼續使用主目錄下的 `class_meeting_record.html`。
* ❌ **不滿意新版本（需要還原）**：
  1. 直接刪除目前有問題的 `class_meeting_record.html`。
  2. 從 `歷史版本` 資料夾中複製您想要還原的舊版本（例如 `v2_...html`）到桌面 `班會紀錄` 資料夾下。
  3. 將該舊版本檔案名稱改回 **`class_meeting_record.html`**，即完成秒速還原！

---

## 📋 規則二：資料草稿備份流程（會議紀錄內容）

網頁上填寫的會議內容（如勾選狀態、輸入的名字與金額）是儲存在瀏覽器的本地空間。為防瀏覽器快取被清理導致紀錄遺失：

1. **草稿防覆蓋複製**：在左側選單的「存檔管理」頁籤中，對已有的草稿點擊 **「複製」** 按鈕，系統會產生一份副本，您可以在副本上進行修改，舊版本草稿便不會被覆蓋。
2. **手動備份 JSON 檔**：完成會議紀錄後，點擊 **「匯出所有紀錄 (.json)」** 下載備份檔。
3. **還原資料**：在新設備或清空快取後，點擊 **「匯入紀錄檔案 (.json)」** 選擇該備份檔，即可找回所有歷史草稿。

---

## 📋 規則三：專業 Git 版本控制還原（雙重保險）

本專案已連結至 Git 版本管理。每一次我們的對話與修改，AI 助理都會自動提交 Git Commit 以防丟失。

如果您想退回到先前的 Git 版本：
1. **查看修改歷史**：在專案資料夾下執行 `git log --oneline`。
2. **提取舊版本**：執行以下指令將特定提交（Commit Hash）的網頁還原：
   ```bash
   git checkout [Commit代碼] -- class_meeting_record.html
   ```
   *例如還原至防切割版：`git checkout 68102ef -- class_meeting_record.html`*

## 📋 規則四：對話討論紀錄留存
* 我們每次對話結束後，AI 助理都會自動將我們全部的討論對話匯出成 **`對話紀錄.html`**。
* 您可以直接點擊網頁頂部的 **「💬 開啟對話紀錄」** 按鈕隨時查閱，或在桌面資料夾直接打開此檔案，對話內容絕不遺失！
